Step 3 of 5

Choose A Project to Host

The first step to hosting a website on GitHub Pages is choosing the site you want to host. In this step, you’ll decide on a site to host. For example, you might choose a website you’ve coded in a previous class. 

The only requirement is that the website can only be powered by HTML, CSS, and/or JavaScript. So, you can’t use a site made using WordPress, for instance. If you’ve never coded an HTML site before, we’ve got you! You can use this example site we’ve coded up(opens in a new tab), to get practice using GitHub Pages. 

After choosing your project, you’ll place the files under version control and then add the repo to your GitHub account. Then, your website is ready for hosting with GitHub Pages! If you need help putting files under version control, refer back to the Create Your Repo step.

WHAT TO DO:
  1. Choose an HTML web project you’ve built previously. For instance, your Jubilee Austen project would be a great choice, if you’ve taken our HTML & CSS class. If you don’t have a site to work with, download the example site(opens in a new tab) by clicking the green Clone or Download button, then Download ZIP. Unzip or extract the files.
  2. Save the website files in a project folder on your computer. Name the folder clearly, with no spaces or uppercase letters, like jubilee-austen-site.
  3. Ensure the website directory is in a logical location, like your computer’s Documents folder. 
  4. Open your text editor and create a new file named .gitignore and save the file in your website’s folder. 
  5. Add .DS_Store and Thumbs.db to the .gitignore file.
  6. Save your changes and close the text editor. 
  7. In the command line, navigate to your project folder.
  8. Get ready to track your website project with Git by writing git init. 
  9. Add all of your files to the repo using git add -A.
  10. Commit your files. Be sure to add a clear and concise commit message. 
  11. Go to GitHub(opens in a new tab) and make sure you’re logged in. Then, create a new repo by clicking the green New button on the left sidebar. 
  12. Name the repo something clear, like Jubilee Austen. Don’t change any other settings on the page and click Create Repository. 
  13. On the next page, copy the commands under the heading that say: “ …or push an existing repository from the command line.” 
  14. In the command line, paste the commands you just copied from GitHub and hit Enter.
  15. Go back to GitHub, hit refresh, and watch your repo appear. You’ll turn this repo into a hosted site in the next challenge. 
  16. When you’re done, copy the link to your repo and submit it below!
